# Python Object-Oriented Programming (OOP)
Welcome to the ultimate Python Object-Oriented Programming repository! This project servers as a step-by-step, file-by-file blueprint designed to transition from a procedural mindset into mastering the core and advaned architecture of Python OOP.

- Phase 1: *Basics & The Foundation*
- Phase 2: *Variable & Method Types*
- Phase 3: *The pillars of OOP*
- Phase 4: *Python-Specific & Advanced*

## 1. Basics & The Foundation
- ### 01_class_and_object.py 
    - Defining a basic class, creating instances, and understanding attributes and methods.
    
- ### 02_init_and_self.py
    - Deep dive into the constructor, passing arguments, and how *self* binds data to an instance.

- ### 03_str_vs_repr.py
    - Overriding dunder methods to control how objects look when printed or dubugged.

## 2. Variable & Method Types
- ### 04_instance_vs_class_vars.py
    - Understanding data that belongs to a sigle object vs. data shared by the entire class.

- ### 05_class_methods.py
    - Using the *@classmethod* decorator (often used in python for alternative constructors).

- ### 06_static_methods.py
    - Using the *@staticmethod* decorator for isolated utility functions.

## 3. The Pillars of OOP
- ### 07_single_inheritance.py
    - Creating a parent class and a child class, and using super().

- ### 08_polymorphism.py
    - Method overriding where different classes use the exact same method name for different behaviors.

- ### 09_encapsulation_private.py
    - Using single '-' and double '--' underscores to hide data and prevent direct modification.

- #### 10_getter_setters.py
    - Using the *@property* decorector to cleanly get, set, and validate private data.

- ### 11_abstraction.py
    - Using the *abc* module to create abstract classes and enforce structure on child classes.
