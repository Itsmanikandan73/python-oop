----------------------

class_methods
-------------
To make a class_method we use "@classmethod" decorator, instead of self( the specific object), it automatically receives cls(the class itself) as its first aurgument.
 
static_method
-------------
self(Instance methods), cls(Class methods), @staticmethod decorator, A static method is completely isolated: it does not receive  self or cls automatically. It is plain , regular  function that just happens to live inside a class beacuse it logically belongs here.
In real world static method are used for "Utility or helper function" they take inputs, do some math or processing, and return a result without touching or changing any class or object data.

single_inheritance
------------------
In c++ we can use : for Inheritance(eg: class dog : public Animal).In python, it's much cleaner: you just pass the parent class as an aurgument into the child class's definition, like class Dog(Animal)
The most critical tool to learn here is "super()" it is a built-in function that lets the chlild class the __init__ constructor or methods of its parent class, ensuring you don't repeat code.

polymorphism
------------
polymorphism comes from greek it means "many forms". In programming language, it means using the exact same function or method name on different object types, and having each object execute its own specific of that method.
In python, polymorphism is incredibly clean and is often implemented via methods overriding(where a child class completely rewrites a method it inherited form a parent class)

encapsulation_private
---------------------
In c++, you use access specifiers like public;, protected; and private: to restrict access to your class data. python does not have strict compiler-enforced access controls.Instead it relies on naming conventions to signal to other developers how data should be treated.
python handles 
    public: variable(access anywhere)
    protected: _variable(A waring to other developers "Please don't touch this outside the class or its subclasses.")
    private: __variable(Trigger "Name mangling" under the hood, making it difficult to access directly from outside the class)

getters_setters
---------------
In previous script we used get_balance() to access the private data.while that works perfectly, python gives us another more elegant way to handle this using @property decorator.
with getters and setters in python,  you can write methods to protect and validate your private attributes, but access them from the outside as if they were regular variables(without using parentheses ().)

abstraction
-----------
In c++ you can create an abstract by decraring "pure virtual function"(eg virtual void draw() = 0). python doesn't have build in virtual keyword.intsead it relies on a build-in module called abc(abstract Base Classes).
An abstract class is a blueprint for other classes. it allows you to declare a method that must exist in any child class, but you leave it empty. "You cannot create an object directly from an abstract class. it enforces a contract on any class that inherits from it.


Phase4 : Advance concepts

multiple_Inheritance
--------------------
Multiple Inheritance means single child class can inherit from more than one parent class. However, this introduces a classic computer science problem called "Diamond Problem"(where two parent classes have a method with the same name, and the child doesn't know which one to run.).
Python solves this problem using an algorithm called MRO (Method Resolution Order). python looks at classes from left to right. depth-first, ensuring every method is called exactly one without crashing.

composition
-----------
In software field there is famous design principle: "Favor object composition over class Inheritance" while Inheritance represents an "ls-A" relationship (Smartphone is Device), compostion represents "Has-A" relationship(a Laptop has a processor, has a Ram stick.).
Instead of forcing a class to inherit a massive tree of code, you build small, specilized class and combine them as components inside a larger object.

operator_overloading
--------------------
In python, operator like +, -, <, and ++ aren't hardcoded to only work with numbers and strings.you can actually teach your custom classes how to respond to these operators!.
This is called Operator Overloading, and it is achieved using pyhton's Dunder(Double underscore) methods (often called "magic methods") when you write a + b a.__add__(b) under  the hood.
